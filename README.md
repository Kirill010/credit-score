# credit-score
# Анализ и прогнозирование оттока клиентов банка

## Описание проекта
Этот проект посвящен анализу факторов, влияющих на отток клиентов банка, и построению модели машинного обучения для прогнозирования вероятности ухода клиента. Такая задача — классический кейс для **продуктового аналитика**: предсказать отток, чтобы вовремя предложить клиенту стимулирующие акции.

**Цель:** Выявить ключевые признаки ухода клиентов и построить модель бинарной классификации (ушел / остался) с высоким качеством предсказания.

## Данные
- **Источник:** [Bank Customer Churn Dataset](https://www.kaggle.com/datasets/sandiledesmondmfazi/bank-customer-churn) (Kaggle)
- **Объем данных:** 115 640 записей о клиентах.
- **Признаки:** 25 атрибутов, включая демографические данные (возраст, пол, доход), поведенческие (срок обслуживания, количество продуктов, баланс), кредитную историю и жалобы.
- **Целевая переменная:** `Churn Flag` (1 — клиент ушел, 0 — остался). Данные несбалансированы (около 12% ушедших).

## Этапы работы

1. **Исследовательский анализ (EDA):**
   - Визуализация распределения целевой переменной (дисбаланс классов).
   - Анализ статистик и сравнение групп клиентов (`groupby` по оттоку).
   - Построение матрицы корреляции для числовых признаков.

2. **Предобработка данных:**
   - Кодирование категориальных признаков (`LabelEncoder`).
   - Удаление неинформативных колонок (`RowNumber`, `CustomerId`, `Surname`, `Churn Reason` и др.).
   - Масштабирование числовых признаков (`StandardScaler`).
   - Разделение данных на train/test (80/20) со стратификацией.

3. **Построение модели:**
   - Выбор модели: **Логистическая регрессия** (базовая и интерпретируемая модель для бинарной классификации).
   - Гиперпараметры: `C=100` (слабая регуляризация), `class_weight='balanced'` (для борьбы с дисбалансом классов).
   - Решатель: `lbfgs`.

4. **Оценка качества:**
   - Метрики: Accuracy, Precision, Recall, F1-score, ROC-AUC, PR-AUC, Log Loss.
   - Построение матрицы ошибок (Confusion Matrix).
   - Визуализация ROC- и PR-кривых, калибровочной кривой.
   - Анализ важности признаков (коэффициенты модели).

> Модель показала выдающиеся результаты (практически идеальную классификацию). Это может говорить как о хорошей предсказательной силе признаков, так и о возможности переобучения или артефактов в данных (например, наличие `Churn Reason` в обучающих данных, хотя он был удален — необходимо перепроверить). Тем не менее, проект демонстрирует полный цикл анализа и построения модели.

### Наиболее важные признаки (по модулю коэффициента)
1. **NumComplaints** — количество жалоб (сильная положительная связь с оттоком)
2. **Credit Score** — кредитный рейтинг
3. **Balance** — баланс на счете
4. **Income** — доход
5. **Outstanding Loans** — сумма непогашенных кредитов

> Отрицательные коэффициенты (например, у `Credit Score`) указывают на то, что рост признака снижает вероятность оттока.

### Визуализации
- **Матрица ошибок:** модель почти не ошибается (единичные ложноположительные срабатывания).
- **ROC-кривая:** AUC = 1.00 — модель идеально разделяет классы.
- **Калибровочная кривая:** вероятности отлично калиброваны (кривая близка к диагонали).
